# 移动端开发最佳实践指南 (Mobile Project Workflow)

本指南教你如何串联 `.trae/Skills` 目录中的各个专家角色，以**工业级标准**从零构建一个高质量的移动应用（Flutter）。

---

## 🛠 移动端开发流 (Mobile Core Workflow)

我们将移动项目的构建分为 6 个阶段，每个阶段都有对应的 Skills 角色支撑。

### 阶段 1：移动端需求与交互头脑风暴
**激活角色**：[01_ProductManager_Brainstorming](.trae/Skills/01_ProductManager_Brainstorming/SKILL.md)
- **目标**：挖掘移动端特有的痛点（如离线使用、定位、推送）。
- **推荐 Prompt**：
  > "作为 **01 号产品经理**，我想做一个移动端[项目名]，请帮我分析移动场景下的核心需求，并重点考虑弱网环境下的用户体验。"

### 阶段 2：跨平台架构与 UI 规范设计
**激活角色**：[02_Architect_APIDesign](.trae/Skills/02_Architect_APIDesign/SKILL.md) & [02_Designer_FrontendImplementation](.trae/Skills/02_Designer_FrontendImplementation/SKILL.md)
- **目标**：定义前后端契约，并确立 iOS/Android 双端适配的设计系统。
- **推荐 Prompt**：
  > "请基于 01 号的需求，由 **02 号架构师** 设计 RESTful 接口；同时请 **02 号设计专家** 参照 Material 3 和 Cupertino 规范，为我定义一套自适应的移动端 UI 组件库。"

### 阶段 3：高质量 Flutter 代码实现
**激活角色**：[03_Mobile_Flutter](.trae/Skills/03_Mobile_Flutter/SKILL.md)
- **目标**：编写符合整洁架构、高性能的 Flutter 代码。
- **推荐 Prompt**：
  > "请激活 **03 号 Flutter 专家**。使用 `Flutter整洁架构` 模式为我实现[功能名]，要求业务逻辑使用 BLoC 封装，并参考 `Flutter性能优化` 指南，确保列表滑动达到 60FPS。"

### 阶段 4：移动端安全与敏感数据保护
**激活角色**：[07_Security_Specialist](.trae/Skills/07_Security_Specialist/SKILL.md)
- **目标**：确保移动端 Token 存储、认证及 GDPR 合规。
- **推荐 Prompt**：
  > "请 **07 号安全专家** 审查我的移动端登录流程。请确保使用 `flutter_secure_storage` 存储敏感信息，并检查数据传输是否符合 GDPR 的安全规范。"

### 阶段 5：自动化测试与真机模拟
**激活角色**：[03_Mobile_Flutter](.trae/Skills/03_Mobile_Flutter/SKILL.md) (模块 4)
- **目标**：通过单元测试、Widget 测试和集成测试降低回归成本。
- **推荐 Prompt**：
  > "请参考 **03 号 Flutter 专家** 的 `Flutter测试与质量保证` 模块，为刚才实现的[功能名]编写 Mocktail 单元测试和 Widget 测试，确保核心交互路径 100% 覆盖。"

### 阶段 6：工程化配置与自动化发布
**激活角色**：[03_Mobile_Flutter](.trae/Skills/03_Mobile_Flutter/SKILL.md) (模块 5) & [05_DevOps_GitOps](.trae/Skills/05_DevOps_GitOps/SKILL.md)
- **目标**：配置多环境 Flavors，并利用 Fastlane 实现自动提审。
- **推荐 Prompt**：
  > "请 **03 号 Flutter 专家** 帮我配置开发/测试/生产三套 Flavors 环境；同时请 **05 号 DevOps 专家** 协助我编写 GitHub Actions 脚本，实现自动打包发布到 TestFlight。"
