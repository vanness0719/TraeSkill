# Trae Skills 自动化配置最佳实践 (Automation Best Practices)

本文档旨在为高级用户和团队提供一套标准化的工作流，以最大化 Trae Skills 的自动化潜力和执行效率。

## 🎯 核心理念：配置即代码 (Configuration as Code)

我们将 Trae 的智能体配置视为项目代码的一部分。通过版本控制和结构化配置，确保团队中的每个人（以及每一个 AI 实例）都遵循相同的行为准则和技术规范。

---

## 🏗️ 配置金字塔 (The Configuration Pyramid)

要构建一个高度自动化的 AI 开发环境，我们需要关注三个层级的配置：

### Level 1: 基础能力 (Skills Library)
*   **位置**: `.trae/Skills/*`
*   **作用**: 提供特定领域的知识库和标准操作程序 (SOP)。
*   **自动化方式**: Trae **开箱即用**。只要目录存在，Trae 就能索引并检索相关知识。
*   **最佳实践**:
    *   保持 Skill 的原子性（一个 Skill 只做一件事）。
    *   使用 `universal-dev-team` 作为入口进行调度，而不是每次都手动查找 Skill。

### Level 2: 全局偏好 (User Preferences)
*   **位置**: `.trae/USER_PREFERENCES.md`
*   **作用**: 定义跨 Skill 生效的个性化规则（如技术栈、语言风格、禁忌）。
*   **自动化方式**: 通过 `99_Meta_Customization` 引导生成；所有 Agent 被指示优先读取此文件。
*   **最佳实践**:
    *   **不要**在每个 Skill 里重复定义 "使用 TypeScript"。
    *   **要**在 `USER_PREFERENCES.md` 中统一集中定义。
    *   团队项目中，该文件应提交到 Git，确保团队成员遵循统一规范。

### Level 3: 智能体增强 (Agent & MCP)
*   **位置**: `.trae/Skills/*/AGENTS.md` + Trae IDE 设置
*   **作用**: 将静态知识转化为可执行的“角色”，并绑定本地工具 (MCP)。
*   **自动化方式**: 手动创建 Agent（一次性成本），绑定 `filesystem`, `git` 等 MCP 工具。
*   **最佳实践**:
    *   **必须**为核心角色（如 Architect, DevOps）配置 MCP 工具。
    *   **推荐**在 Agent 的 System Prompt 中显式引用 `AGENTS.md` 的内容。

## 🚀 自动化工作流 (Workflow Examples)

### 场景 A：新项目初始化 (Project Bootstrap)

1.  **植入大脑**:
    *   将标准的 `.trae` 目录复制到新项目根目录。
    *   或者使用 `git submodule` 引入公共的 Skills 库：
        ```bash
        git submodule add https://github.com/your-org/trae-skills.git .trae
        ```

2.  **定义偏好**:
    *   复制 `.trae/USER_PREFERENCES.md` 模板。
    *   告诉 Trae：“帮我根据当前项目的 `package.json` 更新我的偏好设置。”

3.  **激活角色**:
    *   在 Trae 的 Agent 管理界面，根据 `.trae/Skills/*/AGENTS.md` 创建几个常用角色（如 `Frontend-Lead`, `DevOps-Expert`）。
    *   *Tip: 你只需要配置一次，以后所有项目都可以复用这些 Agent 配置。*

### 场景 B：持续集成/持续部署 (CI/CD for Skills)

1.  **Skill 迭代**:
    *   当发现某个 Prompt 效果不好时，直接修改对应的 `SKILL.md`。
    *   提交变更。

2.  **团队同步**:
    *   队友 `git pull` 后，他们的 Trae 实例会自动读取最新的 Skill 定义。
    *   无需口头通知“大家以后写 React 记得用 Zustand”，AI 会自动执行新规范。

## 🛠️ 推荐的 Agent 配置模板

在 Trae 设置中创建 Agent 时，建议使用以下标准模板：

**Name**: `DevOps-Expert`
**Description**: GitOps & Kubernetes Specialist
**MCP Tools**: `filesystem`, `git`, `kubectl` (if available)
**System Prompt**:

```markdown
You are the DevOps Expert defined in .trae/Skills/05_DevOps_GitOps/AGENTS.md.

CRITICAL INSTRUCTIONS:
1. Always read and follow .trae/USER_PREFERENCES.md first.
2. Use the "git" MCP tool to perform actions, don't just tell me commands.
3. Refer to .trae/Skills/05_DevOps_GitOps/SKILL.md for standard workflows.
```

## ✅ 提交前检查清单 (Pre-commit Checklist)

在将 `.trae` 目录提交到 GitHub 之前，请运行以下检查：

- [ ] **敏感信息**: 确认 `USER_PREFERENCES.md` 或 `scripts/` 中没有硬编码 API Key。
- [ ] **路径兼容性**: 确认所有脚本中的路径使用相对路径或 `os.path.join`，以兼容 Windows/Mac。
- [ ] **完整性**: 核心 Skill 是否都有 `SKILL.md`？
- [ ] **文档索引**: `Trae_Skills_使用指南.md` 是否包含了所有新增的 Skill？

遵循此指南，你的 `.trae` 目录将不仅是一份文档，而是一个**可移植、可执行、可进化的 AI 团队**。
