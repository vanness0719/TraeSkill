# Trae 自然语言唤醒指南 (Natural Language Guide)

Trae 能够通过你的**自然语言意图**自动识别并调用 `.trae/Skills` 中的专家角色。你不需要记住复杂的路径，只需要像对待一个真实的专家团队一样沟通。

---

## ⚡️ 核心唤醒方式

### 1. 全流程“一句话”启动 (最推荐)
如果你有一个新想法，直接说：
> **“我想开发一个 [功能/项目名]，请带我走一遍全流程。”**

**Trae 的反应**：
- 自动激活 [universal-dev-team](./Skills/universal-dev-team/SKILL.md)。
- 扮演 **Team Lead** 梳理节奏。
- 依次调用 **PM -> 架构师 -> 开发 -> QA** 角色。

### 2. 意图驱动的角色自动分配
Trae 会监听特定的关键词来切换专家模式：

| 当你提到... | Trae 会自动进入的角色 |
| :--- | :--- |
| “代码有点乱”、“架构设计” | **02 号架构师** |
| “页面难看”、“UI 不专业” | **02 号设计师** |
| “Flutter”、“手机 App” | **03 号 Flutter 专家** |
| “React”、“Next.js”、“性能慢” | **03 号前端性能专家** |
| “数据库慢”、“SQL 报错” | **05 号数据库专家** |
| “怎么测”、“自动化脚本” | **04 号测试专家** |
| “运营文案”、“推广一下” | **09 号运营专家** |

---

## 💡 让 Trae 更聪明的技巧

### 1. 明确“身份”预期
虽然 Trae 会自动分配，但你可以通过简单的身份指引让它更精准：
- *“作为一名 **安全专家**，看看这段登录逻辑有没有漏洞。”*
- *“用 **Flutter 专家** 的整洁架构模式帮我写这段代码。”*

### 2. 要求“遵循规范”
Trae 的 Skills 库里存储了大量的最佳实践。当你让他写代码时，可以加一句：
- *“请参考你的 **Skills 库规范** 来实现。”*
- *“按 **03 号技能模块** 的性能要求优化这段列表。”*

### 3. “团队”模式下的反馈循环
在 `universal-dev-team` 模式下，Trae 会在每个阶段结束时停下来。你可以说：
- *“PM 的需求我认可了，请进入**架构设计**阶段。”*
- *“代码写完了，现在请 **QA 专家** 帮我写测试用例。”*

---

## 🛠 维护你的“魔咒”
如果你发现 Trae 总是理解错某个意图，你可以：
1. 打开对应的 `SKILL.md`。
2. 在“如何使用”或“触发词”部分添加你习惯的表达方式。
3. Trae 下次就会根据你的个人习惯进行匹配。
