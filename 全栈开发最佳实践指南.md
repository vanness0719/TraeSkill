# Trae 全栈开发最佳实践指南 (Project Workflow Best Practices)

本指南旨在教你如何串联 `.trae/Skills` 目录中的各个专家角色，以**工业级标准**从零构建一个完整的项目。

---

## 🛠 核心开发流 (The Core Workflow)

我们将一个项目的诞生分为 6 个阶段，每个阶段对应特定的 Skills 角色。

### 阶段 1：需求定义与深度头脑风暴
**激活角色**：[01_ProductManager_Brainstorming](.trae/Skills/01_ProductManager_Brainstorming/SKILL.md)
- **目标**：将模糊的想法具象化为功能清单（PRD）。
- **最佳实践**：不要只说“我要做一个外卖 App”，尝试使用**角色指令**。
- **推荐 Prompt**：
  > "作为 **01 号产品经理**，我想做一个针对程序员的健康饮食 App，请帮我进行深度头脑风暴，挖掘核心痛点并生成一份功能优先级清单。"

### 阶段 2：架构设计与接口规范
**激活角色**：[02_Architect_APIDesign](.trae/Skills/02_Architect_APIDesign/SKILL.md) & [02_Designer_WebGuidelines](.trae/Skills/02_Designer_WebGuidelines/SKILL.md) & [02_Designer_FrontendImplementation](.trae/Skills/02_Designer_FrontendImplementation/SKILL.md)
- **目标**：定义技术栈、目录结构和 API 契约。
- **最佳实践**：在写代码前先定“契约”，这能减少前后端联调的痛苦。
- **推荐 Prompt**：
  > "基于 01 号的需求，请 **02 号架构师** 为我设计一套 RESTful API 规范，并给出 Next.js + TailwindCSS 的前端项目目录结构建议。"

### 阶段 3：前端/移动端高质量实现
**激活角色**：[03_Developer_ReactBestPractices](.trae/Skills/03_Developer_ReactBestPractices/SKILL.md) 或 [03_Mobile_Flutter](.trae/Skills/03_Mobile_Flutter/SKILL.md)
- **目标**：编写高性能、符合工程规范的 UI 代码。
- **最佳实践**：开启“性能监控”思维，使用 [03_Developer_ArtifactsBuilder](.trae/Skills/03_Developer_ArtifactsBuilder/SKILL.md) 快速生成组件。
- **推荐 Prompt**：
  > "请使用 **03 号 Flutter 专家** 的整洁架构模式，为我实现登录页面的逻辑，要求 UI 与业务逻辑（BLoC）完全解耦，并参考性能优化指南减少重绘。"

### 阶段 4：稳固的后端与数据库构建
**激活角色**：[05_Backend_Database](.trae/Skills/05_Backend_Database/SKILL.md) & [05_Backend_Python](.trae/Skills/05_Backend_Python/SKILL.md)
- **目标**：构建可扩展的后端服务。
- **最佳实践**：数据库设计必须考虑索引优化和迁移。
- **推荐 Prompt**：
  > "请 **05 号数据库专家** 为我的 App 设计用户表和订单表结构，并提供 SQL 优化建议（如索引设置），同时使用 Python FastAPI 编写核心 CRUD 接口。"

### 阶段 5：安全增强与 AI 能力集成
**激活角色**：[07_Security_Specialist](.trae/Skills/07_Security_Specialist/SKILL.md) & [08_AI_Engineer](.trae/Skills/08_AI_Engineer/SKILL.md)
- **目标**：确保应用安全并引入 LLM 智能化能力。
- **最佳实践**：使用 RAG 提升 AI 的准确性。
- **推荐 Prompt**：
  > "请 **08 号 AI 工程师** 帮我设计一个基于 RAG 的智能食谱推荐系统；同时请 **07 号安全专家** 检查我的认证逻辑是否符合 GDPR 数据保护要求。"

### 阶段 6：自动化测试与质量交付
**激活角色**：[04_Tester_BrowserAutomation](.trae/Skills/04_Tester_BrowserAutomation/SKILL.md)
- **目标**：通过自动化脚本减少回归测试工作量。
- **推荐 Prompt**：
  > "请 **04 号测试专家** 为我的核心下单流程编写 Playwright 自动化测试脚本，确保主路径没有 Bug。"

### 阶段 7：运营增长与数据驱动
**激活角色**：[09_Operations_Growth](.trae/Skills/09_Operations_Growth/SKILL.md)
- **目标**：产出运营文案、分析用户数据并策划增长活动。
- **最佳实践**：内容与数据并重，通过 A/B 测试验证增长假设。
- **推荐 Prompt**：
  > "项目已上线，请 **09 号运营专家** 为我的 App 策划一个『首单立减』的营销活动，并编写 3 条不同侧重点的 App Push 文案以提升转化率。"

## 💡 进阶技巧：万能团队协作 (The Universal Team)

如果你觉得切换角色太麻烦，可以直接启动 [universal-dev-team](.trae/Skills/universal-dev-team/SKILL.md)：

**万能指令**：
> "启动 **universal-dev-team**。现在我们要开发一个[新功能名称]。请按照产品经理 -> 架构师 -> 开发 -> QA 的顺序，一步步带我完成。每一阶段完成后请停下来征求我的反馈。"
